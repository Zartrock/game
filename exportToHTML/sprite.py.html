21   """pygame module with basic game object classes 
22    
23   This module contains several simple classes to be used within games. There 
24   are the main Sprite class and several Group classes that contain Sprites. 
25   The use of these classes is entirely optional when using Pygame. The classes 
26   are fairly lightweight and only provide a starting place for the code 
27   that is common to most games. 
28    
29   The Sprite class is intended to be used as a base class for the different 
30   types of objects in the game. There is also a base Group class that simply 
31   stores sprites. A game could create new types of Group classes that operate 
32   on specially customized Sprite instances they contain. 
33    
34   The basic Sprite class can draw the Sprites it contains to a Surface. The 
35   Group.draw() method requires that each Sprite have a Surface.image attribute 
36   and a Surface.rect. The Group.clear() method requires these same attributes 
37   and can be used to erase all the Sprites with background. There are also 
38   more advanced Groups: pygame.sprite.RenderUpdates() and 
39   pygame.sprite.OrderedUpdates(). 
40    
41   Lastly, this module contains several collision functions. These help find 
42   sprites inside multiple groups that have intersecting bounding rectangles. 
43   To find the collisions, the Sprites are required to have a Surface.rect 
44   attribute assigned. 
45    
46   The groups are designed for high efficiency in removing and adding Sprites 
47   to them. They also allow cheap testing to see if a Sprite already exists in 
48   a Group. A given Sprite can exist in any number of groups. A game could use 
49   some groups to control object rendering, and a completely separate set of 
50   groups to control interaction or player movement. Instead of adding type 
51   attributes or bools to a derived Sprite class, consider keeping the 
52   Sprites inside organized Groups. This will allow for easier lookup later 
53   in the game. 
54    
55   Sprites and Groups manage their relationships with the add() and remove() 
56   methods. These methods can accept a single or multiple group arguments for 
57   membership.  The default initializers for these classes also take a 
58   single group or list of groups as arguments for initial membership. It is safe 
59   to repeatedly add and remove the same Sprite from a Group. 
60    
61   While it is possible to design sprite and group classes that don't derive 
62   from the Sprite and AbstractGroup classes below, it is strongly recommended 
63   that you extend those when you create a new Sprite or Group class. 
64    
65   Sprites are not thread safe, so lock them yourself if using threads. 
66    
67   """

97   class Sprite(object):
98       """simple base class for visible game objects 
99    
100      pygame.sprite.Sprite(*groups): return Sprite 
101   
102      The base class for visible game objects. Derived classes will want to 
103      override the Sprite.update() method and assign Sprite.image and Sprite.rect 
104      attributes.  The initializer can accept any number of Group instances that 
105      the Sprite will become a member of. 
106   
107      When subclassing the Sprite class, be sure to call the base initializer 
108      before adding the Sprite to Groups. 
109   
110      """
111  
112      def __init__(self, *groups):
113          self.__g = {}  # The groups the sprite is in
114          if groups:
115              self.add(*groups)
116  
117      def add(self, *groups):
118          """add the sprite to groups 
119   
120          Sprite.add(*groups): return None 
121   
122          Any number of Group instances can be passed as arguments. The 
123          Sprite will be added to the Groups it is not already a member of. 
124   
125          """
126          has = self.__g.__contains__
127          for group in groups:
128              if hasattr(group, '_spritegroup'):
129                  if not has(group):
130                      group.add_internal(self)
131                      self.add_internal(group)
132              else:
133                  self.add(*group)
134  
135      def remove(self, *groups):
136          """remove the sprite from groups 
137   
138          Sprite.remove(*groups): return None 
139   
140          Any number of Group instances can be passed as arguments. The Sprite 
141          will be removed from the Groups it is currently a member of. 
142   
143          """
144          has = self.__g.__contains__
145          for group in groups:
146              if hasattr(group, '_spritegroup'):
147                  if has(group):
148                      group.remove_internal(self)
149                      self.remove_internal(group)
150              else:
151                  self.remove(*group)
152  
153      def add_internal(self, group):
154          """ 
155          For adding this sprite to a group internally. 
156   
157          :param group: The group we are adding to. 
158          """
159          self.__g[group] = 0
160  
161      def remove_internal(self, group):
162          """ 
163          For removing this sprite from a group internally. 
164   
165          :param group: The group we are removing from. 
166          """
167          del self.__g[group]
168  
169      def update(self, *args, **kwargs):
170          """method to control sprite behavior 
171   
172          Sprite.update(*args, **kwargs): 
173   
174          The default implementation of this method does nothing; it's just a 
175          convenient "hook" that you can override. This method is called by 
176          Group.update() with whatever arguments you give it. 
177   
178          There is no need to use this method if not using the convenience 
179          method by the same name in the Group class. 
180   
181          """
182  
183      def kill(self):
184          """remove the Sprite from all Groups 
185   
186          Sprite.kill(): return None 
187   
188          The Sprite is removed from all the Groups that contain it. This won't 
189          change anything about the state of the Sprite. It is possible to 
190          continue to use the Sprite after this method has been called, including 
191          adding it to Groups. 
192   
193          """
194          for group in self.__g:
195              group.remove_internal(self)
196          self.__g.clear()
197  
198      def groups(self):
199          """list of Groups that contain this Sprite 
200   
201          Sprite.groups(): return group_list 
202   
203          Returns a list of all the Groups that contain this Sprite. 
204   
205          """
206          return list(self.__g)
207  
208      def alive(self):
209          """does the sprite belong to any groups 
210   
211          Sprite.alive(): return bool 
212   
213          Returns True when the Sprite belongs to one or more Groups. 
214          """
215          return truth(self.__g)
216  
217      def __repr__(self):
218          return "<%s Sprite(in %d groups)>" % (self.__class__.__name__,
219                                                len(self.__g))
220  
221      @property
222      def layer(self):
223          """ 
224          Dynamic, read only property for protected _layer attribute. 
225          This will get the _layer variable if it exists. 
226   
227          If you try to get it before it is set it will raise an attribute error. 
228   
229          Layer property can only be set before the sprite is added to a group, 
230          after that it is read only and a sprite's layer in a group should be 
231          set via the group's change_layer() method. 
232   
233          :return: layer as an int, or raise AttributeError. 
234          """
235          return getattr(self, '_layer')
236  
237      @layer.setter
238      def layer(self, value):
239          if not self.alive():
240              setattr(self, '_layer', value)
241          else:
242              raise AttributeError("Can't set layer directly after "
243                                   "adding to group. Use "
244                                   "group.change_layer(sprite, new_layer) "
245                                   "instead.")

248  class DirtySprite(Sprite):
249      """a more featureful subclass of Sprite with more attributes 
250   
251      pygame.sprite.DirtySprite(*groups): return DirtySprite 
252   
253      Extra DirtySprite attributes with their default values: 
254   
255      dirty = 1 
256          If set to 1, it is repainted and then set to 0 again. 
257          If set to 2, it is always dirty (repainted each frame; 
258          flag is not reset). 
259          If set to 0, it is not dirty and therefore not repainted again. 
260   
261      blendmode = 0 
262          It's the special_flags argument of Surface.blit; see the blendmodes in 
263          the Surface.blit documentation 
264   
265      source_rect = None 
266          This is the source rect to use. Remember that it is relative to the top 
267          left corner (0, 0) of self.image. 
268   
269      visible = 1 
270          Normally this is 1. If set to 0, it will not be repainted. (If you 
271          change visible to 1, you must set dirty to 1 for it to be erased from 
272          the screen.) 
273   
274      _layer = 0 
275          0 is the default value but this is able to be set differently 
276          when subclassing. 
277   
278      """
279  
280      def __init__(self, *groups):
281  
282          self.dirty = 1
283  
284          # referred to as special_flags in the documentation of Surface.blit
285          self.blendmode = 0
286          self._visible = 1
287  
288          # Default 0 unless initialized differently.
289          self._layer = getattr(self, '_layer', 0)
290          self.source_rect = None
291          Sprite.__init__(self, *groups)
292  
293      def _set_visible(self, val):
294          """set the visible value (0 or 1) and makes the sprite dirty"""
295          self._visible = val
296          if self.dirty < 2:
297              self.dirty = 1
298  
299      def _get_visible(self):
300          """return the visible value of that sprite"""
301          return self._visible
302  
303      @property
304      def visible(self):
305          """ 
306          You can make this sprite disappear without removing it from the group 
307          assign 0 for invisible and 1 for visible 
308          """
309          return self._get_visible()
310  
311      @visible.setter
312      def visible(self, value):
313          self._set_visible(value)
314  
315      @property
316      def layer(self):
317          """ 
318          Layer property can only be set before the sprite is added to a group, 
319          after that it is read only and a sprite's layer in a group should be 
320          set via the group's change_layer() method. 
321   
322          Overwrites dynamic property from sprite class for speed. 
323          """
324          return self._layer
325  
326      @layer.setter
327      def layer(self, value):
328          if not self.alive():
329              self._layer = value
330          else:
331              raise AttributeError("Can't set layer directly after "
332                                   "adding to group. Use "
333                                   "group.change_layer(sprite, new_layer) "
334                                   "instead.")
335  
336      def __repr__(self):
337          return "<%s DirtySprite(in %d groups)>" % \
338              (self.__class__.__name__, len(self.groups()))

341  class AbstractGroup(object):
342      """base class for containers of sprites 
343   
344      AbstractGroup does everything needed to behave as a normal group. You can 
345      easily subclass a new group class from this or the other groups below if 
346      you want to add more features. 
347   
348      Any AbstractGroup-derived sprite groups act like sequences and support 
349      iteration, len, and so on. 
350   
351      """
352  
353      # dummy val to identify sprite groups, and avoid infinite recursion
354      _spritegroup = True
355  
356      def __init__(self):
357          self.spritedict = {}
358          self.lostsprites = []
359  
360      def sprites(self):
361          """get a list of sprites in the group 
362   
363          Group.sprite(): return list 
364   
365          Returns an object that can be looped over with a 'for' loop. (For now, 
366          it is always a list, but this could change in a future version of 
367          pygame.) Alternatively, you can get the same information by iterating 
368          directly over the sprite group, e.g. 'for sprite in group'. 
369   
370          """
371          return list(self.spritedict)
372  
373      def add_internal(self,
374                       sprite,
375                       layer=None  # noqa pylint: disable=unused-argument; supporting legacy derived classes that override in non-pythonic way
376                       ):
377          """ 
378          For adding a sprite to this group internally. 
379   
380          :param sprite: The sprite we are adding. 
381          :param layer: the layer to add to, if the group type supports layers 
382          """
383          self.spritedict[sprite] = 0
384  
385      def remove_internal(self, sprite):
386          """ 
387          For removing a sprite from this group internally. 
388   
389          :param sprite: The sprite we are removing. 
390          """
391          lost_rect = self.spritedict[sprite]
392          if lost_rect:
393              self.lostsprites.append(lost_rect)
394          del self.spritedict[sprite]
395  
396      def has_internal(self, sprite):
397          """ 
398          For checking if a sprite is in this group internally. 
399   
400          :param sprite: The sprite we are checking. 
401          """
402          return sprite in self.spritedict
403  
404      def copy(self):
405          """copy a group with all the same sprites 
406   
407          Group.copy(): return Group 
408   
409          Returns a copy of the group that is an instance of the same class 
410          and has the same sprites in it. 
411   
412          """
413          return self.__class__(self.sprites()) # noqa pylint: disable=too-many-function-args; needed because copy() won't work on AbstractGroup
414  
415      def __iter__(self):
416          return iter(self.sprites())
417  
418      def __contains__(self, sprite):
419          return self.has(sprite)
420  
421      def add(self, *sprites):
422          """add sprite(s) to group 
423   
424          Group.add(sprite, list, group, ...): return None 
425   
426          Adds a sprite or sequence of sprites to a group. 
427   
428          """
429          for sprite in sprites:
430              # It's possible that some sprite is also an iterator.
431              # If this is the case, we should add the sprite itself,
432              # and not the iterator object.
433              if isinstance(sprite, Sprite):
434                  if not self.has_internal(sprite):
435                      self.add_internal(sprite)
436                      sprite.add_internal(self)
437              else:
438                  try:
439                      # See if sprite is an iterator, like a list or sprite
440                      # group.
441                      self.add(*sprite)
442                  except (TypeError, AttributeError):
443                      # Not iterable. This is probably a sprite that is not an
444                      # instance of the Sprite class or is not an instance of a
445                      # subclass of the Sprite class. Alternately, it could be an
446                      # old-style sprite group.
447                      if hasattr(sprite, '_spritegroup'):
448                          for spr in sprite.sprites():
449                              if not self.has_internal(spr):
450                                  self.add_internal(spr)
451                                  spr.add_internal(self)
452                      elif not self.has_internal(sprite):
453                          self.add_internal(sprite)
454                          sprite.add_internal(self)
455  
456      def remove(self, *sprites):
457          """remove sprite(s) from group 
458   
459          Group.remove(sprite, list, or group, ...): return None 
460   
461          Removes a sprite or sequence of sprites from a group. 
462   
463          """
464          # This function behaves essentially the same as Group.add. It first
465          # tries to handle each argument as an instance of the Sprite class. If
466          # that fails, then it tries to handle the argument as an iterable
467          # object. If that fails, then it tries to handle the argument as an
468          # old-style sprite group. Lastly, if that fails, it assumes that the
469          # normal Sprite methods should be used.
470          for sprite in sprites:
471              if isinstance(sprite, Sprite):
472                  if self.has_internal(sprite):
473                      self.remove_internal(sprite)
474                      sprite.remove_internal(self)
475              else:
476                  try:
477                      self.remove(*sprite)
478                  except (TypeError, AttributeError):
479                      if hasattr(sprite, '_spritegroup'):
480                          for spr in sprite.sprites():
481                              if self.has_internal(spr):
482                                  self.remove_internal(spr)
483                                  spr.remove_internal(self)
484                      elif self.has_internal(sprite):
485                          self.remove_internal(sprite)
486                          sprite.remove_internal(self)
487  
488      def has(self, *sprites):
489          """ask if group has a sprite or sprites 
490   
491          Group.has(sprite or group, ...): return bool 
492   
493          Returns True if the given sprite or sprites are contained in the 
494          group. Alternatively, you can get the same information using the 
495          'in' operator, e.g. 'sprite in group', 'subgroup in group'. 
496   
497          """
498          if not sprites:
499              return False  # return False if no sprites passed in
500  
501          for sprite in sprites:
502              if isinstance(sprite, Sprite):
503                  # Check for Sprite instance's membership in this group
504                  if not self.has_internal(sprite):
505                      return False
506              else:
507                  try:
508                      if not self.has(*sprite):
509                          return False
510                  except (TypeError, AttributeError):
511                      if hasattr(sprite, '_spritegroup'):
512                          for spr in sprite.sprites():
513                              if not self.has_internal(spr):
514                                  return False
515                      else:
516                          if not self.has_internal(sprite):
517                              return False
518  
519          return True
520  
521      def update(self, *args, **kwargs):
522          """call the update method of every member sprite 
523   
524          Group.update(*args, **kwargs): return None 
525   
526          Calls the update method of every member sprite. All arguments that 
527          were passed to this method are passed to the Sprite update function. 
528   
529          """
530          for sprite in self.sprites():
531              sprite.update(*args, **kwargs)
532  
533      def draw(self, surface):
534          """draw all sprites onto the surface 
535   
536          Group.draw(surface): return None 
537   
538          Draws all of the member sprites onto the given surface. 
539   
540          """
541          sprites = self.sprites()
542          if hasattr(surface, "blits"):
543              self.spritedict.update(
544                  zip(
545                      sprites,
546                      surface.blits((spr.image, spr.rect) for spr in sprites)
547                  )
548              )
549          else:
550              for spr in sprites:
551                  self.spritedict[spr] = surface.blit(spr.image, spr.rect)
552          self.lostsprites = []
553  
554      def clear(self, surface, bgd):
555          """erase the previous position of all sprites 
556   
557          Group.clear(surface, bgd): return None 
558   
559          Clears the area under every drawn sprite in the group. The bgd 
560          argument should be Surface which is the same dimensions as the 
561          screen surface. The bgd could also be a function which accepts 
562          the given surface and the area to be cleared as arguments. 
563   
564          """
565          if callable(bgd):
566              for lost_clear_rect in self.lostsprites:
567                  bgd(surface, lost_clear_rect)
568              for clear_rect in self.spritedict.values():
569                  if clear_rect:
570                      bgd(surface, clear_rect)
571          else:
572              surface_blit = surface.blit
573              for lost_clear_rect in self.lostsprites:
574                  surface_blit(bgd, lost_clear_rect, lost_clear_rect)
575              for clear_rect in self.spritedict.values():
576                  if clear_rect:
577                      surface_blit(bgd, clear_rect, clear_rect)
578  
579      def empty(self):
580          """remove all sprites 
581   
582          Group.empty(): return None 
583   
584          Removes all the sprites from the group. 
585   
586          """
587          for sprite in self.sprites():
588              self.remove_internal(sprite)
589              sprite.remove_internal(self)
590  
591      def __nonzero__(self):
592          return truth(self.sprites())
593  
594      __bool__ = __nonzero__
595  
596      def __len__(self):
597          """return number of sprites in group 
598   
599          Group.len(group): return int 
600   
601          Returns the number of sprites contained in the group. 
602   
603          """
604          return len(self.sprites())
605  
606      def __repr__(self):
607          return "<%s(%d sprites)>" % (self.__class__.__name__, len(self))

610  class Group(AbstractGroup):
611      """container class for many Sprites 
612   
613      pygame.sprite.Group(*sprites): return Group 
614   
615      A simple container for Sprite objects. This class can be subclassed to 
616      create containers with more specific behaviors. The constructor takes any 
617      number of Sprite arguments to add to the Group. The group supports the 
618      following standard Python operations: 
619   
620          in      test if a Sprite is contained 
621          len     the number of Sprites contained 
622          bool    test if any Sprites are contained 
623          iter    iterate through all the Sprites 
624   
625      The Sprites in the Group are not ordered, so the Sprites are drawn and 
626      iterated over in no particular order. 
627   
628      """
629      def __init__(self, *sprites):
630          AbstractGroup.__init__(self)
631          self.add(*sprites)

638  class RenderUpdates(Group):
639      """Group class that tracks dirty updates 
640   
641      pygame.sprite.RenderUpdates(*sprites): return RenderUpdates 
642   
643      This class is derived from pygame.sprite.Group(). It has an enhanced draw 
644      method that tracks the changed areas of the screen. 
645   
646      """
647      def draw(self, surface):
648          surface_blit = surface.blit
649          dirty = self.lostsprites
650          self.lostsprites = []
651          dirty_append = dirty.append
652          for sprite in self.sprites():
653              old_rect = self.spritedict[sprite]
654              new_rect = surface_blit(sprite.image, sprite.rect)
655              if old_rect:
656                  if new_rect.colliderect(old_rect):
657                      dirty_append(new_rect.union(old_rect))
658                  else:
659                      dirty_append(new_rect)
660                      dirty_append(old_rect)
661              else:
662                  dirty_append(new_rect)
663              self.spritedict[sprite] = new_rect
664          return dirty

667  class OrderedUpdates(RenderUpdates):
668      """RenderUpdates class that draws Sprites in order of addition 
669   
670      pygame.sprite.OrderedUpdates(*spites): return OrderedUpdates 
671   
672      This class derives from pygame.sprite.RenderUpdates().  It maintains 
673      the order in which the Sprites were added to the Group for rendering. 
674      This makes adding and removing Sprites from the Group a little 
675      slower than regular Groups. 
676   
677      """
678      def __init__(self, *sprites):
679          self._spritelist = []
680          RenderUpdates.__init__(self, *sprites)
681  
682      def sprites(self):
683          return list(self._spritelist)
684  
685      def add_internal(self, sprite, layer=None):
686          RenderUpdates.add_internal(self, sprite)
687          self._spritelist.append(sprite)
688  
689      def remove_internal(self, sprite):
690          RenderUpdates.remove_internal(self, sprite)
691          self._spritelist.remove(sprite)

694  class LayeredUpdates(AbstractGroup):
695      """LayeredUpdates Group handles layers, which are drawn like OrderedUpdates 
696   
697      pygame.sprite.LayeredUpdates(*spites, **kwargs): return LayeredUpdates 
698   
699      This group is fully compatible with pygame.sprite.Sprite. 
700      New in pygame 1.8.0 
701   
702      """
703  
704      _init_rect = Rect(0, 0, 0, 0)
705  
706      def __init__(self, *sprites, **kwargs):
707          """initialize an instance of LayeredUpdates with the given attributes 
708   
709          You can set the default layer through kwargs using 'default_layer' 
710          and an integer for the layer. The default layer is 0. 
711   
712          If the sprite you add has an attribute _layer, then that layer will be 
713          used. If **kwarg contains 'layer', then the passed sprites will be 
714          added to that layer (overriding the sprite._layer attribute). If 
715          neither the sprite nor **kwarg has a 'layer', then the default layer is 
716          used to add the sprites. 
717   
718          """
719          self._spritelayers = {}
720          self._spritelist = []
721          AbstractGroup.__init__(self)
722          self._default_layer = kwargs.get('default_layer', 0)
723  
724          self.add(*sprites, **kwargs)
725  
726      def add_internal(self, sprite, layer=None):
727          """Do not use this method directly. 
728   
729          It is used by the group to add a sprite internally. 
730   
731          """
732          self.spritedict[sprite] = self._init_rect
733  
734          if layer is None:
735              try:
736                  layer = sprite.layer
737              except AttributeError:
738                  layer = self._default_layer
739                  setattr(sprite, '_layer', layer)
740          elif hasattr(sprite, '_layer'):
741              setattr(sprite, '_layer', layer)
742  
743          sprites = self._spritelist  # speedup
744          sprites_layers = self._spritelayers
745          sprites_layers[sprite] = layer
746  
747          # add the sprite at the right position
748          # bisect algorithmus
749          leng = len(sprites)
750          low = mid = 0
751          high = leng - 1
752          while low <= high:
753              mid = low + (high - low) // 2
754              if sprites_layers[sprites[mid]] <= layer:
755                  low = mid + 1
756              else:
757                  high = mid - 1
758          # linear search to find final position
759          while mid < leng and sprites_layers[sprites[mid]] <= layer:
760              mid += 1
761          sprites.insert(mid, sprite)

763      def add(self, *sprites, **kwargs):
764          """add a sprite or sequence of sprites to a group 
765   
766          LayeredUpdates.add(*sprites, **kwargs): return None 
767   
768          If the sprite you add has an attribute _layer, then that layer will be 
769          used. If **kwarg contains 'layer', then the passed sprites will be 
770          added to that layer (overriding the sprite._layer attribute). If 
771          neither the sprite nor **kwarg has a 'layer', then the default layer is 
772          used to add the sprites. 
773   
774          """
775  
776          if not sprites:
777              return
778          layer = kwargs['layer'] if 'layer' in kwargs else None
779          for sprite in sprites:
780              # It's possible that some sprite is also an iterator.
781              # If this is the case, we should add the sprite itself,
782              # and not the iterator object.
783              if isinstance(sprite, Sprite):
784                  if not self.has_internal(sprite):
785                      self.add_internal(sprite, layer)
786                      sprite.add_internal(self)
787              else:
788                  try:
789                      # See if sprite is an iterator, like a list or sprite
790                      # group.
791                      self.add(*sprite, **kwargs)
792                  except (TypeError, AttributeError):
793                      # Not iterable. This is probably a sprite that is not an
794                      # instance of the Sprite class or is not an instance of a
795                      # subclass of the Sprite class. Alternately, it could be an
796                      # old-style sprite group.
797                      if hasattr(sprite, '_spritegroup'):
798                          for spr in sprite.sprites():
799                              if not self.has_internal(spr):
800                                  self.add_internal(spr, layer)
801                                  spr.add_internal(self)
802                      elif not self.has_internal(sprite):
803                          self.add_internal(sprite, layer)
804                          sprite.add_internal(self)

807          """Do not use this method directly. 
808   
809          The group uses it to add a sprite.

831      def draw(self, surface):
832          """draw all sprites in the right order onto the passed surface 
833   
834          LayeredUpdates.draw(surface): return Rect_list 
835   
836          """
837          spritedict = self.spritedict
838          surface_blit = surface.blit
839          dirty = self.lostsprites
840          self.lostsprites = []
841          dirty_append = dirty.append
842          init_rect = self._init_rect
843          for spr in self.sprites():
844              rec = spritedict[spr]
845              newrect = surface_blit(spr.image, spr.rect)
846              if rec is init_rect:
847                  dirty_append(newrect)
848              else:
849                  if newrect.colliderect(rec):
850                      dirty_append(newrect.union(rec))
851                  else:
852                      dirty_append(newrect)
853                      dirty_append(rec)
854              spritedict[spr] = newrect
855          return dirty

900          """change the layer of the sprite

903   
904          The sprite must have been added to the renderer already. This is not 
905          checked.

908          sprites = self._spritelist  # speedup
909          sprites_layers = self._spritelayers  # speedup

922                  low = mid + 1
923              else:
924                  high = mid - 1

927              mid += 1

930              setattr(sprite, '_layer', new_layer)

1028     """LayeredDirty Group is for DirtySprites; subclasses LayeredUpdates 
1029  
1030     pygame.sprite.LayeredDirty(*spites, **kwargs): return LayeredDirty 
1031  
1032     This group requires pygame.sprite.DirtySprite or any sprite that 
1033     has the following attributes: 
1034         image, rect, dirty, visible, blendmode (see doc of DirtySprite). 
1035  
1036     It uses the dirty flag technique and is therefore faster than 
1037     pygame.sprite.RenderUpdates if you have many static sprites.  It 
1038     also switches automatically between dirty rect updating and full 
1039     screen drawing, so you do no have to worry which would be faster. 
1040  
1041     As with the pygame.sprite.Group, you can specify some additional attributes 
1042     through kwargs: 
1043         _use_update: True/False   (default is False) 
1044         _default_layer: default layer where the sprites without a layer are 
1045             added 
1046         _time_threshold: threshold time for switching between dirty rect mode 
1047             and fullscreen mode; defaults to updating at 80 frames per second, 
1048             which is equal to 1000.0 / 80.0 
1049  
1050     New in pygame 1.8.0 
1051  
1052     """
1053 
1054     def __init__(self, *sprites, **kwargs):
1055         """initialize group. 
1056  
1057         pygame.sprite.LayeredDirty(*spites, **kwargs): return LayeredDirty 
1058  
1059         You can specify some additional attributes through kwargs: 
1060             _use_update: True/False   (default is False) 
1061             _default_layer: default layer where the sprites without a layer are 
1062                 added 
1063             _time_threshold: threshold time for switching between dirty rect 
1064                 mode and fullscreen mode; defaults to updating at 80 frames per 
1065                 second, which is equal to 1000.0 / 80.0 
1066  
1067         """
1068         LayeredUpdates.__init__(self, *sprites, **kwargs)
1069         self._clip = None
1070 
1071         self._use_update = False
1072 
1073         self._time_threshold = 1000.0 / 80.0  # 1000.0 / fps
1074 
1075         self._bgd = None
1076         for key, val in kwargs.items():
1077             if (key in ['_use_update', '_time_threshold', '_default_layer']
1078                     and hasattr(self, key)):

1083

1089             raise AttributeError()
1090         if not hasattr(sprite, 'visible'):
1091             raise AttributeError()
1092         if not hasattr(sprite, 'blendmode'):
1093             raise AttributeError()
1094

1096             raise TypeError()

1103     def draw(self, surface, bgd=None):  # noqa pylint: disable=arguments-differ; unable to change public interface
1104         """draw all sprites in the right order onto the given surface 
1105  
1106         LayeredDirty.draw(surface, bgd=None): return Rect_list 
1107  
1108         You can pass the background too. If a self.bgd is already set to some 
1109         value that is not None, then the bgd argument has no effect. 
1110  
1111         """
1112         # functions and classes assigned locally to speed up loops
1113         orig_clip = surface.get_clip()
1114         latest_clip = self._clip
1115         if latest_clip is None:
1116             latest_clip = orig_clip
1117 
1118         local_sprites = self._spritelist
1119         local_old_rect = self.spritedict
1120         local_update = self.lostsprites
1121         rect_type = Rect
1122 
1123         surf_blit_func = surface.blit
1124         if bgd is not None:
1125             self._bgd = bgd
1126         local_bgd = self._bgd
1127 
1128         surface.set_clip(latest_clip)
1129         # -------
1130         # 0. decide whether to render with update or flip
1131         start_time = get_ticks()
1132         if self._use_update:  # dirty rects mode
1133             # 1. find dirty area on screen and put the rects into
1134             # self.lostsprites still not happy with that part
1135             self._find_dirty_area(latest_clip, local_old_rect,
1136                                   rect_type, local_sprites,
1137                                   local_update,
1138                                   local_update.append, self._init_rect)
1139             # can it be done better? because that is an O(n**2) algorithm in
1140             # worst case
1141 
1142             # clear using background
1143             if local_bgd is not None:
1144                 for rec in local_update:
1145                     surf_blit_func(local_bgd, rec, rec)
1146 
1147             # 2. draw
1148             self._draw_dirty_internal(local_old_rect, rect_type,
1149                                       local_sprites,
1150                                       surf_blit_func, local_update)
1151             local_ret = list(local_update)
1152         else:  # flip, full screen mode
1153             if local_bgd is not None:
1154                 surf_blit_func(local_bgd, (0, 0))
1155             for spr in local_sprites:
1156                 if spr.visible:
1157                     local_old_rect[spr] = surf_blit_func(spr.image,
1158                                                          spr.rect,
1159                                                          spr.source_rect,
1160                                                          spr.blendmode)
1161             # return only the part of the screen changed
1162             local_ret = [rect_type(latest_clip)]
1163 
1164         # timing for switching modes
1165         # How may a good threshold be found? It depends on the hardware.
1166         end_time = get_ticks()
1167         if end_time-start_time > self._time_threshold:
1168             self._use_update = False
1169         else:
1170             self._use_update = True
1171 
1172         # emtpy dirty rects list
1173         local_update[:] = []
1174 
1175         # -------
1176         # restore original clip
1177         surface.set_clip(orig_clip)
1178         return local_ret

1180     @staticmethod
1181     def _draw_dirty_internal(_old_rect, _rect, _sprites, _surf_blit,
1182                              _update):
1183         for spr in _sprites:
1184             if spr.dirty < 1 and spr.visible:
1185                 # sprite not dirty; blit only the intersecting part
1186                 if spr.source_rect is not None:
1187                     # For possible future speed up, source_rect's data
1188                     # can be pre-fetched outside of this loop.
1189                     _spr_rect = _rect(spr.rect.topleft,
1190                                       spr.source_rect.size)
1191                     rect_offset_x = spr.source_rect[0] - _spr_rect[0]
1192                     rect_offset_y = spr.source_rect[1] - _spr_rect[1]
1193                 else:
1194                     _spr_rect = spr.rect
1195                     rect_offset_x = -_spr_rect[0]
1196                     rect_offset_y = -_spr_rect[1]
1197 
1198                 _spr_rect_clip = _spr_rect.clip
1199 
1200                 for idx in _spr_rect.collidelistall(_update):
1201                     # clip
1202                     clip = _spr_rect_clip(_update[idx])
1203                     _surf_blit(spr.image,
1204                                clip,
1205                                (clip[0] + rect_offset_x,
1206                                 clip[1] + rect_offset_y,
1207                                 clip[2],
1208                                 clip[3]),
1209                                spr.blendmode)
1210             else:  # dirty sprite
1211                 if spr.visible:
1212                     _old_rect[spr] = _surf_blit(spr.image,
1213                                                 spr.rect,
1214                                                 spr.source_rect,
1215                                                 spr.blendmode)
1216                 if spr.dirty == 1:
1217                     spr.dirty = 0

1219     @staticmethod
1220     def _find_dirty_area(_clip, _old_rect, _rect, _sprites, _update,
1221                          _update_append, init_rect):
1222         for spr in _sprites:
1223             if spr.dirty > 0:
1224                 # chose the right rect
1225                 if spr.source_rect:
1226                     _union_rect = _rect(spr.rect.topleft,
1227                                         spr.source_rect.size)
1228                 else:
1229                     _union_rect = _rect(spr.rect)
1230 
1231                 _union_rect_collidelist = _union_rect.collidelist
1232                 _union_rect_union_ip = _union_rect.union_ip
1233                 i = _union_rect_collidelist(_update)
1234                 while i > -1:
1235                     _union_rect_union_ip(_update[i])
1236                     del _update[i]
1237                     i = _union_rect_collidelist(_update)
1238                 _update_append(_union_rect.clip(_clip))
1239 
1240                 if _old_rect[spr] is not init_rect:
1241                     _union_rect = _rect(_old_rect[spr])
1242                     _union_rect_collidelist = _union_rect.collidelist
1243                     _union_rect_union_ip = _union_rect.union_ip
1244                     i = _union_rect_collidelist(_update)
1245                     while i > -1:
1246                         _union_rect_union_ip(_update[i])
1247                         del _update[i]
1248                         i = _union_rect_collidelist(_update)
1249                     _update_append(_union_rect.clip(_clip))

1473 def collide_circle(left, right):
1474     """detect collision between two sprites using circles 
1475  
1476     pygame.sprite.collide_circle(left, right): return bool 
1477  
1478     Tests for collision between two sprites by testing whether two circles 
1479     centered on the sprites overlap. If the sprites have a "radius" attribute, 
1480     then that radius is used to create the circle; otherwise, a circle is 
1481     created that is big enough to completely enclose the sprite's rect as 
1482     given by the "rect" attribute. This function is intended to be passed as 
1483     a collided callback function to the *collide functions. Sprites must have a 
1484     "rect" and an optional "radius" attribute. 
1485  
1486     New in pygame 1.8.0 
1487  
1488     """
1489 
1490     xdistance = left.rect.centerx - right.rect.centerx
1491     ydistance = left.rect.centery - right.rect.centery
1492     distancesquared = xdistance ** 2 + ydistance ** 2
1493 
1494     try:
1495         leftradius = left.radius
1496     except AttributeError:
1497         leftrect = left.rect
1498         # approximating the radius of a square by using half of the diagonal,
1499         # might give false positives (especially if its a long small rect)
1500         leftradius = (0.5 * ((leftrect.width ** 2 +
1501                               leftrect.height ** 2) ** 0.5))
1502         # store the radius on the sprite for next time
1503         left.radius = leftradius
1504 
1505     try:
1506         rightradius = right.radius
1507     except AttributeError:
1508         rightrect = right.rect
1509         # approximating the radius of a square by using half of the diagonal
1510         # might give false positives (especially if its a long small rect)
1511         rightradius = (0.5 * ((rightrect.width ** 2 +
1512                                rightrect.height ** 2) ** 0.5))
1513         # store the radius on the sprite for next time
1514         right.radius = rightradius
1515     return distancesquared <= (leftradius + rightradius) ** 2
